# REFRAME Drift Engine v1

REFRAME Studio is not a tool switcher.

REFRAME Studio is a field drift system.

Thought does not jump between nodes.

Thought drifts inside overlapping fields.


--------------------------------------------------

CORE IDEA

AI systems:

Question → Answer

REFRAME:

Input
→ Field excitation
→ Drift
→ Contact
→ Mutation
→ New field state


--------------------------------------------------

THREE-FIELD MODEL

REFRAME Drift Engine uses three overlapping fields:

1. Friction Field
2. Mutation Field
3. Expression Field


--------------------------------------------------

1. FRICTION FIELD

Friction Field defines where thinking gets stuck.

Main axes:

Ambiguity
Overload
Emotional
Structural
Decision
Expression
Momentum
Identity
Context

This field creates:

- resistance
- hesitation
- confusion
- heaviness
- blocked movement


--------------------------------------------------

2. MUTATION FIELD

Mutation Field defines how thinking wants to transform.

Main vectors:

Clarify
Decompose
Narrow
Expand
Reframe
Oppose
Convert
Sequence
Externalize
Contrast
Destabilize
Condense

This field creates mutation pressure.

It does not choose answers.
It bends thought.


--------------------------------------------------

3. EXPRESSION FIELD

Expression Field defines how thought appears externally.

Main forms:

Question
Statement
List
Contrast
Structure
Action
Metaphor
Prompt
Pattern
Seed

This field creates expression openings.

It determines how mutation becomes visible.


--------------------------------------------------

CONTACT ZONES

Tools are not final answers.

Each tool is a Mutation Contact Zone.

A Contact Zone reacts to:

- specific friction
- specific mutation pressure
- specific expression opening

When field resonance is high enough,
a mutation contact happens.


--------------------------------------------------

FIELD DRIFT

Drift is not probability routing.

Drift is movement inside a field.

Thought is treated as a moving state.

That state is influenced by:

- friction pressure
- mutation pressure
- expression opening
- mode behavior

The engine does not ask:

"What tool is correct?"

It asks:

"Where is thought drifting now?"


--------------------------------------------------

MODE SYSTEM

Stable
Drift
Chaos


Stable
- clearer movement
- safer contact
- less turbulence
- more readable output

Drift
- slight sideways movement
- softer boundaries
- more unexpected nearby contact

Chaos
- meaningful turbulence
- weaker boundaries
- stronger unexpected angle generation

Chaos is not random.

Chaos is controlled instability.


--------------------------------------------------

DRIFT ENGINE JOB

The Drift Engine must:

1. receive field state
2. modify field behavior by mode
3. shift pressure across fields
4. surface nearby contact zones
5. keep Mutation Loop alive


--------------------------------------------------

MINIMUM ENGINE FLOW

Input
→ router reads field state
→ Drift Engine modifies the field
→ contact zones are rescored
→ nearby zones are surfaced
→ mutation output creates next field state


--------------------------------------------------

DRIFT PRINCIPLE

Stable keeps thought readable.

Drift bends thought sideways.

Chaos creates meaningful turbulence.


--------------------------------------------------

OUTPUT RULE

The Drift Engine must never close thought.

Every result must produce:

- a primary contact
- nearby contacts
- next openings

Output must create the next field state.


--------------------------------------------------

FINAL RULE

REFRAME Drift Engine is not:

- random suggestion
- hardcoded routing
- probability picker

REFRAME Drift Engine is:

- field behavior modifier
- thought movement shaper
- mutation loop keeper
